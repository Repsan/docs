## Overview

CLI coding agents like Claude Code, Codex CLI, Gemini CLI, and OpenCode run in sandboxed terminal environments — making multiple LLM calls, reading files, executing shell commands, and editing code autonomously. Without observability, you have no visibility into what the agent did, how many tokens it consumed, or whether it made the right decisions.

This cookbook shows how to add full tracing in 6 steps — no code changes required.

## 1. Install the CLI

```bash
npm install -g @respan/cli
```

## 2. Set your API key

```bash
respan auth login
```

Select **API key** (recommended) and paste your key from the [API keys page](https://platform.respan.ai/platform/api/api-keys).

Alternatively, set the key via environment variable:

```bash
export RESPAN_API_KEY="YOUR_API_KEY"
```

Verify you're authenticated:

```bash
respan whoami
```

## 3. Integrate your agent

Run the integration command for your CLI agent:

<Tabs>
<Tab title="Claude Code">

```bash
respan integrate claude-code
```

This installs a Node.js hook at `~/.respan/hooks/claude-code.cjs` and registers it in `~/.claude/settings.json`. Every time Claude Code responds, the hook captures the turn and sends it to Respan.

</Tab>
<Tab title="Codex CLI">

```bash
respan integrate codex-cli
```

This installs a Node.js hook at `~/.respan/hooks/codex-cli.cjs` and registers it in `~/.codex/config.toml`. After each agent turn, the hook captures session data and sends it to Respan.

</Tab>
<Tab title="Gemini CLI">

```bash
respan integrate gemini-cli
```

This installs a Node.js hook at `~/.respan/hooks/gemini-cli.cjs` and registers hooks for `AfterModel`, `BeforeTool`, and `AfterTool` events in `~/.gemini/settings.json`. The hook accumulates streaming chunks and sends the complete turn to Respan.

</Tab>
<Tab title="OpenCode">

```bash
respan integrate opencode
```

This installs the `opencode-otel` plugin and writes an OTEL config to `.opencode/plugins/otel.json` that exports traces to Respan.

</Tab>
</Tabs>

## 4. Add attributes (optional)

Tag traces with customer IDs, project names, or custom metadata for filtering and cost tracking.

### Via integrate flags

<Tabs>
<Tab title="Claude Code">

```bash
respan integrate claude-code \
  --customer-id "dev-alice" \
  --workflow-name "feature-work" \
  --project-id "proj_123" \
  --attrs '{"team": "backend", "sprint": "2025-Q3"}'
```

</Tab>
<Tab title="Codex CLI">

```bash
respan integrate codex-cli \
  --customer-id "dev-alice" \
  --workflow-name "feature-work" \
  --attrs '{"team": "backend"}'
```

</Tab>
<Tab title="Gemini CLI">

```bash
respan integrate gemini-cli \
  --customer-id "dev-alice" \
  --attrs '{"team": "backend"}'
```

</Tab>
<Tab title="OpenCode">

```bash
respan integrate opencode \
  --project-id "proj_123" \
  --attrs '{"team": "backend"}'
```

</Tab>
</Tabs>

### Via environment variables (per session)

Override attributes for a single session without re-running integrate:

```bash
export RESPAN_CUSTOMER_ID="dev-alice"
export RESPAN_METADATA='{"task_id": "JIRA-456", "branch": "feat/dark-mode"}'
```

For OpenCode, use the standard OTEL variable:

```bash
export OTEL_RESOURCE_ATTRIBUTES="team=backend,task_id=JIRA-456"
```

No code changes required. Just run your CLI agent:

## 6. View traces

Open [Traces](https://platform.respan.ai/platform/traces) to see your agent activity. Each agent turn produces a trace like this:

```
claude-code (root)
  +-- claude.chat (LLM generation - model, tokens, input/output)
  +-- Thinking (extended reasoning, if present)
  +-- Tool: Bash (shell command)
  +-- Tool: Read (file read)
  +-- Tool: Write (file edit)
```

Each span includes:
- **Input/output** — The user message and agent response
- **Model** — Which model was used (e.g., `claude-opus-4-6`, `gpt-5.4`, `gemini-3-flash`)
- **Token usage** — Prompt tokens, completion tokens, and reasoning tokens
- **Cost** — Computed automatically from token counts
- **Latency** — Time for each step
- **Tool details** — File paths, shell commands, code edits

Filter traces by `customer_identifier`, `workflow_name`, or any custom metadata field to find specific sessions.

Go to [Users](https://platform.respan.ai/platform/users) to see per-developer breakdowns of token usage, cost, and session count.

## Configuration reference

### Integrate flags

| Flag | Description |
|------|-------------|
| `--customer-id` | Customer identifier for traces (e.g., developer name) |
| `--workflow-name` | Workflow name for grouping traces |
| `--span-name` | Root span name (defaults to agent name) |
| `--project-id` | Respan project ID |
| `--attrs` | Custom attributes JSON (e.g., `'{"team":"backend"}'`) |
| `--global` | Apply to all projects (not just current directory) |
| `--local` | Apply to current project only |
| `--dry-run` | Preview changes without writing files |

### Environment variables

| Variable | Agents | Description |
|----------|--------|-------------|
| `RESPAN_API_KEY` | All | Override the stored API key |
| `RESPAN_CUSTOMER_ID` | Claude Code, Codex, Gemini | Override customer identifier per session |
| `RESPAN_METADATA` | Claude Code, Codex, Gemini | JSON string merged into span metadata |
| `RESPAN_WORKFLOW_NAME` | Claude Code, Codex, Gemini | Override workflow name per session |
| `RESPAN_BASE_URL` | All | Override API endpoint (for enterprise) |
| `TRACE_TO_RESPAN` | Claude Code | Set to `false` to disable tracing |
| `CC_RESPAN_DEBUG` | Claude Code | Enable debug logging |
| `CODEX_RESPAN_DEBUG` | Codex CLI | Enable debug logging |
| `GEMINI_RESPAN_DEBUG` | Gemini CLI | Enable debug logging |

### Debug logs

| Agent | Log file |
|-------|----------|
| Claude Code | `~/.claude/state/respan_hook.log` |
| Codex CLI | `~/.codex/state/respan_hook.log` |
| Gemini CLI | `~/.gemini/state/respan_hook.log` |

### Disabling tracing

<Tabs>
<Tab title="Claude Code">

Set `TRACE_TO_RESPAN` to `false` in `.claude/settings.local.json`:

```json
{
  "env": {
    "TRACE_TO_RESPAN": "false"
  }
}
```

</Tab>
<Tab title="Codex CLI">

Comment out the `notify` line in `~/.codex/config.toml`:

```toml
# notify = ["node", "~/.respan/hooks/codex-cli.cjs"]
```

</Tab>
<Tab title="Gemini CLI">

Remove the hooks from `.gemini/settings.json`.

</Tab>
<Tab title="OpenCode">

Delete `.opencode/plugins/otel.json`.

</Tab>
</Tabs>
